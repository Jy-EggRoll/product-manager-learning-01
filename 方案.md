一、项目背景
1.1 业务背景
某医药零售连锁品牌旗下门店分布于多个城市核心商圈及社区，现面临两大核心经营压力：
• 线下客流持续下滑，线上电商与社区团购不断蚕食份额，单店坪效承压；
• 沉淀的会员数据庞大但未被激活，"人找药"仍是主要服务模式，主动触达能力弱。
领导层核心诉求：在存量会员中挖出增量销售，将会员资产真正盘活，同时实现"感知-决策-执行-反馈"全链路自动化闭环，打造可复制的数字化转型标杆。
1.2 战略对齐
本项目紧密对齐业务高层的核心预期，以“增量销售、会员 LTV 提升、坪效提升、全链路自动化”为四大核心战略方向，其中，增量销售聚焦从“人找药”到“药找人”的模式转变，通过精准运营盘活会员资产；会员 LTV 提升则重点围绕慢性病患者复购周期管理和季节性疾病提前拦截展开，延长会员生命周期价值；坪效提升依托智能推荐功能，优化门店转化效率；全链路自动化则致力于实现“感知-决策-执行-反馈”的全自动闭环，彻底摆脱对人工的过度依赖。
1.3 用户价值

1. 从“单店试点”到“全国推广”的复制成本趋零
本产品跑通的慢病复购周期、天气/舆情触发等核心模型，本质是一套“感知-决策-触达-履约”的自动化规则引擎。一旦在50家试点门店验证有效（复购率提升、坪效增长），向全国数千家门店推广时：
无需增加总部运营人员，策略由智能体自动适配区域差异（如南方台风、北方暴雪触发不同用药关怀）。
门店店长接受的“每日任务包”式交互极易上手，培训成本极低，适合大规模推广。
2. 从“直营连锁”到“加盟赋能”的商业模式升级
对于大型医药零售企业，这套系统不仅是内部工具，更可包装为对加盟店的“智慧赋能包”：
加盟商缴纳服务费，即可获得“总部级”的智能营销能力、补货建议和个性化触达工具。
总部从“抽成/供货”关系升级为“技术服务+供应链优化”的赋能平台，打开新的营收来源。
3. 从“医药零售”到“泛零售”的跨行业借鉴
产品核心模型（外部信号感知→用户画像匹配→智能策略生成→多通道触达）具备极强的行业迁移能力：
便利店/商超：天气触发鲜食折扣、库存消化。
母婴店：按宝宝月龄自动推送辅食、奶粉复购。
宠物店：驱虫药周期复购、季节性疾病预防。 这套方法论沉淀下来，将成为企业对外输出数字化转型咨询和技术方案的基石。

3.2 智能体定位
本项目中的智能体定位为“数字副店长”，具备感知、判断、执行、学习四大核心能力，能够将一线人员的实操经验沉淀为标准化、自动化的运营策略，实现经验的复用和优化。在协作模式上，总部运营人员的角色将从传统的“做活动”转变为“养策略”，智能体负责活动的实时执行、数据监测等具体工作，运营人员则专注于策略审核、规则调优和效果复盘；对于门店店长和店员而言，智能体相当于一个“外挂大脑”，每天会推送今日重点会员名单、补货建议和服务话术提示，助力一线人员高效开展工作，提升服务质量和转化效率。

四、业务场景
4.1 智能营销
场景一：季节性预防营销
在花粉季、流感季、高温高发期来临前，智能体自动识别易感人群并推送关联药品及防护用品优惠。如过敏性鼻炎高发前两周，结合历史用药数据、花粉浓度预报和会员过敏标签，圈选高风险会员，生成含抗组胺药、鼻喷剂、防护口罩的组合优惠券并自动触达，提醒提前备药预防。
场景二：天气触发营销
当监测到某城市即将出现极端天气时，智能体在预警发出后4小时内，对区域内特定慢病会员发起精准关怀和营销。如寒潮来临前，针对心脑血管慢病会员推送“护心关怀券包”，含药品复购券和血压计折扣券；高温预警城市则推送防暑套装优惠，并附健康贴士，提升打开率与转化率。
场景三：舆情热点营销
当本地出现“流感爆发”等健康热搜时，智能体自动抓取舆情，为热点区域门店周边会员推送针对性防护方案。如某城市登上“甲流”热搜，2小时内筛选出有儿童且近期未购奥司他韦的家庭，推送“家庭防疫应急包”满减券，消化抗病毒药和口罩库存，同时联动库存模块对不足门店发出补货提醒。
4.2 会员复购
场景一：慢病长期用药复购提醒
智能体为每位慢病会员建立独立用药周期档案，记录购药时间、规格和购买量，自动计算药品耗尽日。在耗尽前数天，通过会员偏好渠道推送复购提醒和专属优惠券，让触达恰逢其需，提升转化率，避免断药造成的健康风险与营收流失。
场景二：个性周期动态调整与关联推荐
智能体根据会员实际购药间隔行为动态调整个人化触达窗口，避免一刀切。在会员启动复购时，结合健康标签和消费历史，推荐关联品类如血糖试纸、护肝保健品等，借场景化组合提升客单价，传递专业懂我的印象，促进会员升级。
场景三：复购失效自动挽回与流失预警
会员收到复购提醒后72小时未核销，系统自动进入挽回流程：二次触达更换文案并可能提高优惠力度。连续两次错过则标记为“复购高危”，推送店长进行人工回访，探查断药原因并记录反馈，形成“自动挽回+人工兜底”的闭环，最大化降低流失率。
4.3 智能库存优化
场景一：日常动态补货建议
每天营业前，智能体综合近期销售、库存、在途订单和未来销量预测，生成“今日建议补货清单”并附简要逻辑说明。店长可一键确认或微调数量，将精力更多投入会员服务，同时补货科学性大幅提升。
场景二：季节性备货与营销活动备货
智能模块拉取历史同期销售数据和当年天气、流行病预测，在季节到来前数周提醒店长逐步备货。当智能营销模块发起主题大促时，系统联动进行“活动库存体检”，对库存不足支撑预估流量的门店紧急提醒补足，确保营销资源不浪费，会员体验不受损。
场景三：突发事件驱动的紧急补货与调拨
本地发生流感爆发、高温预警等事件时，系统立即启动需求评估，扫描所有门店相关药品库存，对低于安全水位的门店发出紧急补货提醒，并计算中心仓或邻近门店可调拨库存，生成最优调拨方案，抓住突发需求机遇，避免缺货损失。
场景四：滞销风险预警与动态出清
系统监控每个SKU动销情况，在库时间超过平均周转天数1.5倍时自动标记“滞销预警”，结合效期和营销资源，智能建议出清方案（如买赠、定向满减券），店长审核后一键执行，减少报损并回拢资金，形成全周期库存管理闭环。
五、系统技术架构
5.1 四层架构设计
本系统采用四层架构设计，从上至下依次为接入层、应用层、能力层和数据层，各层级分工明确、协同联动，确保系统稳定高效运行。接入层作为用户交互的入口，涵盖运营管理后台、店长工作台、企微侧边栏、会员小程序和 API 开放平台，满足不同角色的使用需求；应用层是系统的核心执行层，包含智能体核心引擎、策略规则引擎、活动编排引擎、触达渠道管理和数据分析中心，负责实现智能体的核心功能；能力层为系统提供技术支撑，涵盖会员画像服务、药品知识图谱、外部信号感知、推荐算法服务和自然语言处理，保障系统的智能化水平；数据层则负责数据的存储和管理，包含会员数据库、药品数据库、策略数据库、活动数据库和日志分析库，为系统运行提供数据支撑。

5.2 各层详细说明
接入层的各组件各司其职，运营管理后台主要用于策略配置管理、标签规则维护、券模板管理和活动复盘分析，是总部运营人员的核心操作平台；店长工作台为门店店长提供今日任务清单、补货建议确认、会员触达记录和销售业绩查看等功能，助力店长高效管理门店日常运营；企微侧边栏聚焦一线店员的实操需求，实时展示会员画像、用药周期提醒、智能推荐话术和券包快捷发送功能，提升店员服务效率；会员小程序面向会员用户，支持推送消息查看、券包领取、用药提醒和健康管理，增强会员粘性；API 开放平台则用于第三方系统对接和数据接口开放，提升系统的兼容性和扩展性。
应用层是智能体功能落地的核心，智能体核心引擎负责实现感知-洞察-决策的完整闭环，具备多模态理解、上下文记忆管理和自学习迭代优化能力；策略规则引擎负责会员与药品的匹配规则管理、触发条件判定、冲突解决策略制定以及 A/B 测试支持，确保策略的科学性和灵活性；活动编排引擎负责活动的自动生成、执行追踪和状态管理，实现活动全流程自动化；触达渠道管理负责企微、短信、小程序等多渠道的管理和推送策略优化，提升信息触达效率；数据分析中心则负责转化分析、效果评估和报表生成，为策略优化提供数据支撑。
能力层为系统提供全方位的技术支撑，会员画像服务负责构建会员基础属性标签、记录用药历史、分析消费行为和评估健康风险，为精准运营提供依据；药品知识图谱涵盖药品分类、适应症、禁忌症、关联推荐和配伍禁忌等内容，保障用药推荐的专业性和安全性；外部信号感知负责季节节气监测、天气预报对接、舆情热点监控和地理位置服务，确保系统能够实时捕捉外部动态；推荐算法服务采用协同过滤、关联规则、个性化推荐等算法，结合冷启动处理，提升推荐的精准度；自然语言处理则负责话术生成、意图识别、情感分析和对话管理，保障智能交互的流畅性和专业性。
数据层存储系统运行所需的各类数据，会员数据库存储会员基础信息、用药记录、消费记录和健康档案，为会员画像构建提供数据支撑；药品数据库存储药品基本信息、分类标签、库存信息和价格信息，保障药品相关功能的正常运行；策略数据库存储规则配置、标签定义、券模板和活动配置，支撑策略的落地执行；活动数据库存储活动实例、触达记录、转化数据和核销记录，用于活动效果复盘；日志分析库存储操作日志、行为日志、异常日志和审计日志，为系统维护和问题排查提供依据。
5.3 关键技术特点
本系统的关键技术特点体现在五个方面，其一的全链路自动化闭环，实现了感知-决策-执行-反馈-学习全流程的自动化运行，无需人工过多干预，大幅提升运营效率；其二是实时感知外部信号，能够对季节、天气、舆情、地理位置等外部动态进行实时监测，确保活动能够及时响应市场变化；其三是智能体自学习迭代能力，系统能够根据活动转化数据持续优化策略参数，不断提升运营精准度；其四是多端协同触达，整合企微、短信、小程序等多渠道资源，实现对目标会员的全方位、精准触达；其五是数据驱动决策，所有运营策略均基于数据分析制定，活动效果可量化评估，确保策略的科学性和有效性。

七、MVP（最小可行产品）范围与成功指标
7.1 MVP功能范围
为确保项目快速验证和迭代，MVP阶段聚焦以下核心能力：
核心闭环跑通：先完成慢病复购周期和天气触发两大场景的自动化闭环。
关键角色覆盖：上线“总部运营-智能策略中心”和“店长-每日任务包”两个核心交互界面。
试点城市与门店：选择5家门店作为试点，跑通数据、流程和效果。
核心数据接入：完成会员交易数据、慢病标签、部分药品库存数据和外部天气数据的接入。
7.2 核心衡量指标（成功标准）
业务增长指标：
增量销售额：因智能体发起的自动化活动产生的销售额（可设置对照门店进行对比）。
会员复购率：在复购周期管理场景下，慢病会员的复购比例提升。
效率提升指标：
活动上线耗时：从捕捉信号到活动上线，从“周”级降至“小时”级。
库存周转天数：试点门店在西药/中成药上的库存周转天数优化。
